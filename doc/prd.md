# 退休服务工具 MVP 产品需求文档

## 一、 文档概述
### 1.1 产品定位
一款面向 45-62 岁人群的**个人退休资产管理与政策导航工具**。核心价值在于：将复杂的“延迟退休”政策转化为直观的“时间表”，将晦涩的“社保条文”转化为结构化的“行动建议”。
### 1.2 核心原则 (MVP)
* **隐私第一**：数据本地存储（LocalStorage/IndexedDB），无需注册登录。
* **计算驱动**：以 2026 年新规为基准的精准退休模拟。
* **去社交化**：专注个人工具属性，无评论、无点赞、无好友。

---

## 二、 核心功能模块

### 2.1 延迟退休精准计算器 (Logic Core)
**功能描述**：基于 2025 年 1 月 1 日起施行的渐进式延迟法定退休年龄政策，计算用户的精确退休时间。
* **输入项**：
    * 出生年月（Picker）
    * 性别（Male / Female）
    * 原退休档位（男 60、女干部 55、女工人 50）
* **输出项**：
    * **原法定退休日期** vs **新法定退休日期**
    * **延迟月数**
    * **倒计时**（距离退休还有多少天/月）
* **技术逻辑**：
    * 需内置中国《关于实施渐进式延迟法定退休年龄的决定》的换算公式。
    * $T_{new} = T_{old} + \Delta m$（其中 $\Delta m$ 根据出生日期跨度线性增长，封顶 3 年或 5 年）。

### 2.2 政策知识库 (RAG/Static Query)
**功能描述**：提供本地化、通俗化的社保/医保政策指引。
* **核心功能**：
    * **地区筛选**：默认展示国家通用政策，支持选择特定省份（如：北京、上海、深圳）。
    * **卡片化展示**：将政策拆解为“养老金上调”、“医保报销”、“工龄认定”等模块。
    * **极简搜索**：支持关键字检索（建议后续结合用户的 RAG 经验，使用语义搜索）。
* **技术要求**：
    * 数据源以 `JSON` 格式存储，包含 `title`, `summary`, `tags`, `region` 字段。

### 2.3 个人退休“三步走”计划 (Plan Generator)
**功能描述**：根据用户年龄自动生成的行动清单（ToDo List）。
* **阶段划分**：
    1.  **筹备期（退休前 5 年）**：核查社保缴费年限、补缴咨询、档案位置确认。
    2.  **办理期（退休前 6 个月）**：材料清单、办理地点引导。
    3.  **领取期（退休后）**：待遇资格认证提醒、医保个人账户激活。
* **状态跟踪**：支持用户手动勾选完成状态，进度持久化在浏览器。

## 三、 技术架构与数据规范

### 3.1 技术栈建议
* **框架**：Next.js (App Router) + Tailwind CSS。
* **状态管理**：Zustand（轻量，适合持久化存储用户计算结果）。
* **UI 组件库**：Radix UI / Shadcn UI（易于 Claude Code 编写高对比度组件）。

### 3.2 数据结构 (Data Schema)
```typescript
interface UserProfile {
  birthDate: string; // YYYY-MM
  gender: 'male' | 'female';
  type: 'worker' | 'cadre'; // 身份类型
  location: string;
  insuranceYears: number; // 已缴费年限
}

interface PolicyCard {
  id: string;
  category: 'pension' | 'medical' | 'process';
  title: string;
  content: string; // Markdown format
  isPinned: boolean;
}
```

## 四、 UI/UX 设计规范 (适老化专项)
* **视觉规范**：
    * **字号**：正文不小于 **18px**，标题 **24px+**。
    * **对比度**：符合 WCAG AA 标准，黑/深蓝底白字或白底黑字。
    * **间距**：按钮热区高度至少 **48px**，避免误触。
* **交互规范**：
    * **零弹窗**：所有信息在当前页切换或展开，避免多层级 Modal 导致的迷失感。
    * **触觉反馈**：点击按钮时提供明显的视觉缩放反馈（`active:scale-95`）。

## 五、 开发路线图 (Claude Code 实施步序)

1.  **Step 1**: 实现基于 2025 新规的退休日期换算逻辑函数（纯数学逻辑测试）。
2.  **Step 2**: 构建用户基础信息录入表单（Big Input/Big Button）。
3.  **Step 3**: 开发“退休倒计时”与“计划卡片”组件。
4.  **Step 4**: 导入 Mock 政策数据 JSON，实现分类筛选功能。
5.  **Step 5**: 接入 LocalStorage 插件，确保用户刷新页面后数据不丢失。
